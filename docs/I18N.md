# Internationalization

> Maintenance workflows (sync after content changes, adding a locale,
> terminology verification) are scripted step-by-step in the project skill
> `.claude/skills/i18n/SKILL.md` — use `/i18n` in Claude Code.

The app has two independent language settings (Tools → Language):

- **Interface** — UI chrome, via Flutter gen_l10n (`lib/l10n/app_<code>.arb`).
- **Game content** — data names (clans, skills, techniques, items, conditions,
  …), via a display-time overlay (`assets/i18n/data_<code>.json`).

Content defaults to matching the interface; either can be set to English
independently. English data names remain the canonical identifiers
everywhere — lookups, rules logic (`lib/rules_constants.dart`), and save
files never contain translated strings. Translation happens only at render
time through `trData()` / `dataL10n.trCondition()` (`lib/data_l10n.dart`),
which fall back to the English string for any missing entry. A missing or
malformed overlay collapses to pure English; it can never crash the app.

## Robustness invariants

- Overlay entries are `"English name": "translation"`; empty/non-string
  values are dropped on load.
- `test/i18n_sync_test.dart` hard-fails CI if an overlay key no longer
  matches a translatable data string (e.g. after a data-audit rename):
  update `assets/i18n/data_<code>.json` in the same commit. Missing
  translations are *not* failures — new content simply displays in English
  (the coverage report in that test is informational).
- The authoritative set of translatable strings is
  `lib/i18n_harvest.dart`; the sync test dumps it to
  `build/l10n_harvest.json` for tooling.
- User-provided content (homebrew, rules descriptions) is never translated;
  it displays as authored via the identity fallback. Imports decode as
  UTF-8 with graceful handling of malformed bytes.

## Adding a locale

1. `lib/l10n/app_<code>.arb` — copy `app_en.arb`'s keys and translate
   (`flutter gen-l10n` regenerates; missing keys fall back to English).
2. Add the locale to `supportedUiLocales` / `supportedContentCodes` in
   `lib/locale_controllers.dart`, and to the pickers in
   `lib/screens/tools_page.dart`.
3. `assets/i18n/data_<code>.json` — flat map of English data string →
   translation. Run `flutter test test/i18n_sync_test.dart` (produces the
   harvest), then `python3 scripts/import_i18n.py <code>` — it seeds from
   the upstream Qt CSV when one exists (de/es are ~87% complete).
4. Add the locale to the loops in `test/i18n_sync_test.dart` and mirror
   `test/i18n_fr_smoke_test.dart`.

French typography conventions used: typographic apostrophe (’), no-break
space before `: ; ! ?` where practical, gender-neutral message templates
("Supprimé : {name}").

## German sources

The German 5e rulebook ("Die Legende der fünf Ringe – Das Rollenspiel",
Fantasy Flight Games / Asmodee) exists, but no freely available official
term list or reference sheet was found (checked 2026-07). The locale is
therefore **not book-verified**: it keeps the upstream volunteer CSV's
terms where they existed (~47% of strings; volunteers likely worked from
the printed book — e.g. Zwist, Fähigkeit, Lehrplan) and careful new
translation for the rest. Treat corrections from owners of the German
books as authoritative and apply them as one-line overlay edits.

## Spanish sources

The Spanish 5e rulebook ("La Leyenda de los Cinco Anillos: El juego de
rol", Edge Entertainment España) exists, but no freely available official
term list was found (checked 2026-07). The upstream volunteer CSV is
notably clean and consistent (Escuela de … pattern, proper genitives) and
likely follows the printed book; its terms were kept for the ~47% of
strings it covered. The locale is still **not book-verified** — treat
corrections from owners of the Spanish books as authoritative one-line
overlay edits. App-side rules vocabulary used: NO (número objetivo) for
TN, conflicto for strife, PX for XP, DJ for GM.

## French sources

Terminology was verified against official Edge French terms where a
reference exists (see README credits): conditions and core-book technique
names (Kata, Kihō, Shūji, Rituels) follow the reference lists maintained by
La Voix de Rokugan. Everything else (Invocations, Mahō, Ninjutsu, item and
trait names beyond the upstream CSV, book-expansion content) is translated
but not book-verified — treat corrections from owners of the French books
as authoritative and apply them as one-line overlay edits.
